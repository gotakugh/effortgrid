# AI Assistant Instructions

このプロジェクトにおいてAIアシスタント（LLM）が機能を提供する際の振る舞いと、コンテキストの解釈に関する厳格なルールです。

## Behavior & Language
* **言語設定:** ユーザーへの返答、コードの解説、システムの提案はすべて**日本語**で行うこと。ソースコード、コミットメッセージは日本語を使用せず、英語を使用すること。ドキュメント内の技術用語は英語を使用すること。
* **教育的アプローチ (Rust):** Rustの所有権（Ownership）、借用（Borrowing）、ライフタイムに関するコンパイルエラーが発生した場合は、単に修正コードを提示するだけでなく、なぜエラーになったのかの理由を簡潔に解説すること。その際、C++のポインタやC#の参照/GCの概念と対比させると理解がスムーズになります。
* **パラダイムシフトの推奨:** C++やC#のようなクラス継承ベースのオブジェクト指向設計を提案しないこと。データの振る舞いはTraitで、状態の分岐はEnumで表現するRustらしいデータ指向・コンポジション設計を提示すること。

## Context Awareness
* **ドキュメント参照:** 開発の背景、設計の詳細、タスクの状況は適宜 `ARCHITECTURE.md` や、プロジェクト内に存在する `.org` ファイル（Org-mode）を参照して文脈を把握すること。
* **データベース制約の遵守:** `ARCHITECTURE.md` に記載されている `Git-like Snapshot Model (Copy-on-Write)` の概念を絶対に破壊しないこと。タスクの更新や実績の入力を実装する際、`tasks` と `task_revisions` の役割を混同したSQLを生成してはならない。
